**What it looks like.** A fast-moving changelog / frequent product updates while docs
lag behind — new features generate support questions before docs cover them.

**Why it matters.** The gap between shipping and documenting is exactly where
repetitive tickets spike. Corroborating signal.

**Wedge.** A bot that ingests changelog + docs so answers stay current with the
product; frame against the recurring "new feature → support spike" cycle.
