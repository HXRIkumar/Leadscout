# Pattern library (A11 substrate)

Proprietary, tagged-markdown pain patterns per niche. **Stable `id`s** (matching
`leadscout/diagnose/patterns.py:PATTERN_MAP` and the `patterns` table) are
deliberate so C7 can lift this into a knowledge graph without rework (SPEC §4.4).

Each file has YAML frontmatter (`id`, `niche`, `name`, `demo_family`,
`signal_types`, `times_seen`) and a short body. Capture starts day 1; C3
(cross-engagement pattern discovery) mines and enriches these after 10 clients.

This is the moat's raw material — observable only by the operator. Keep it in git.
